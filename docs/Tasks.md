## Roadmap και Καταμερισμός Εργασίας (To-Do List)

Ο φόρτος εργασίας πρέπει να επιμεριστεί έτσι ώστε **και οι δύο μας** να ασχοληθούμε με τη συγγραφή κώδικα για τη λειτουργικότητα της εφαρμογής, όχι μόνο με τα εικαστικά.

### Φάση 1: Θεμελίωση (Κληρονομικότητα, SGG, Βασική Δομή)

| Task | Υπεύθυνος | Σκοπός / Σημειώσεις |
| :--- | :--- | :--- |
| **1.1. Setup & SGG** | Μηχανικός Γραφικών | Εγκατάσταση του project, ενσωμάτωση SGG, ρύθμιση του βασικού βρόγχου `main` και του παραθύρου. |
| **1.2. Βάση Κλάσης & GlobalState** | Μηχανικός Λογικής | Δήλωση της κλάσης `GlobalState` (Singleton) και των βασικών κλάσεων `VisualAsset` και `Node`. Υλοποίηση των μεθόδων `init()`, `update()`, `draw()` στην `GlobalState`. |
| **1.3. Ιεραρχία Οντοτήτων** | Μηχανικός Λογικής | Δημιουργία της ιεραρχίας: `VisualAsset` -> `Node/Station`, `Train`, `Passenger`. Εξασφάλιση πολυμορφικής κλήσης των `draw()` και `update()`,. |
| **1.4. Δεδομένα Γράφου** | Μηχανικός Γραφικών | Δημιουργία των αρχείων δεδομένων (π.χ. JSON/XML - *επιτρέπονται εξωτερικές βιβλιοθήκες για διάβασμα αρχείων δεδομένων*) για τις συντεταγμένες των σταθμών (Nodes) και τις αποστάσεις/βάρη (Edges). |

### Φάση 2: Λογική (Γράφος, Δρομολόγηση, Συλλογές)

| Task | Υπεύθυνος | Σκοπός / Σημειώσεις |
| :--- | :--- | :--- |
| **2.1. Δομή Γράφου** | Μηχανικός Λογικής | Υλοποίηση της δομής του γράφου (π.χ. λίστα γειτνίασης) μέσα στην `GlobalState` ή σε σχετική κλάση. Χρήση **STL συλλογών** (`std::map`, `std::vector`) για αποθήκευση των `Node` και `Edge`. |
| **2.2. Αλγόριθμος Δρομολόγησης** | Μηχανικός Λογικής | Υλοποίηση του αλγορίθμου για τον υπολογισμό της βέλτιστης διαδρομής (sequence of stations) για τους `Passenger`. |
| **2.3. Προσομοίωση Κίνησης** | Μηχανικός Λογικής | Υλοποίηση της λογικής κίνησης των `Train` κατά μήκος των γραμμών (Edges) και της διαδικασίας παραλαβής/αποβίβασης `Passenger`. |
| **2.4. Δυναμική Διαχείριση** | Μηχανικός Λογικής | Υλοποίηση της δυναμικής δημιουργίας `Passenger` (νέα ζήτηση) και της καταστροφής τους μόλις φτάσουν στον προορισμό τους (χρήση `new` και `delete`). |

---
**Υπενθύμιση:** Η επιτυχία της εφαρμογής βασίζεται στο να λειτουργεί η **Προσομοίωση δρομολόγησης κίνησης δικτύου**. Η διαχείριση του μετρό είναι σαν τη διαχείριση ενός δικτύου ύδρευσης, όπου αντί για νερό, μεταφέρουμε επιβάτες, και πρέπει να ρυθμίζουμε τη ροή (συχνότητα συρμών) για να αποφύγουμε την υπερχείλιση (overcrowding).
