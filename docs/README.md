| Χαρακτηριστικό | Περιγραφή | Σχετική Απαίτηση Πηγών |
| :--- | :--- | :--- |
| **Τίτλος** | Athens Metro Manager | - |
| **Θεματολογία** | Διαδραστική εφαρμογή με χρήση δικτυώματος (γράφου) από κόμβους (σταθμοί μετρό). | |
| **Βασικός Στόχος** | Διαχείριση της κυκλοφορίας και της συχνότητας των συρμών ώστε να αποφευχθεί η υπερφόρτωση (overcrowding) των σταθμών, ικανοποιώντας τη ζήτηση μεταφοράς επιβατών. Αυτό εμπίπτει στην κατηγορία «Προσομοίωση δρομολόγησης κίνησης δικτύου» (αυξημένος βαθμός δυσκολίας). | |
| **Βαθμολογία** | Βασίζεται στον αριθμό των επιβατών που μεταφέρθηκαν επιτυχώς σε σχέση με τον μέσο χρόνο αναμονής. Όλα τα δεδομένα, στατιστικά και σκορ φυλάσσονται στην κλάση `GlobalState`. | |
| **Γραφική Έξοδος** | Υποχρεωτική χρήση της βιβλιοθήκης **Simple Graphics Library (SGG)**,. | |

## B. Βασικά Components και Υλοποίηση (Classes & Concepts)

Η αρχιτεκτονική πρέπει να βασίζεται αυστηρά στην **κληρονομικότητα** και τον **πολυμορφισμό**,.

| Component / Κλάση | Ρόλος και Υλοποίηση | Σχετική Απαίτηση |
| :--- | :--- | :--- |
| **1. GlobalState** | Το μοναδικό στιγμιότυπο που φυλάει την κατάσταση (level, σκορ, raw δεδομένα προσομοίωσης). Περιέχει τις βασικές μεθόδους `init()`, `update()`, και `draw()`, οι οποίες καλούν τις αντίστοιχες μεθόδους για όλα τα αντικείμενα του παιχνιδιού. Παρέχει επίσης κεντρικό σημείο πρόσβασης σε δεδομένα (π.χ. `getWindowSize()`). | Υποχρεωτική Κλάση Κατάστασης,. |
| **2. VisualAsset** | Η γενική κλάση βάσης για όλα τα οπτικά στοιχεία (ό,τι εμφανίζεται επί της οθόνης). Περιέχει τις πολυμορφικές μεθόδους `draw()` και `update()`. | Κληρονομικότητα/Πολυμορφισμός,. |
| **3. Node / Station** | Οι σταθμοί του μετρό (οι κόμβοι του γράφου). Κληρονομεί από την `VisualAsset`. Αποθηκεύει τη γεωγραφική θέση, τη σύνδεση με άλλους κόμβους (Edge), και τον τρέχοντα φόρτο (λίστα `Passenger` – χρήση STL συλλογής),. | Δίκτυο Κόμβων. |
| **4. Passenger** | Αντιπροσωπεύει έναν επιβάτη. Είναι μια οντότητα που δημιουργείται δυναμικά στη μνήμη (χρήση `new`). Έχει προορισμό και πρέπει να υπολογίζει τη βέλτιστη διαδρομή χρησιμοποιώντας έναν αλγόριθμο (π.χ., A-star algorithm). | Δυναμική Μνήμη, Δρομολόγηση. |
| **5. Train** | Αντιπροσωπεύει τον συρμό. Κινείται κατά μήκος των ακμών. Επίσης δυναμική οντότητα. | Δυναμική Μνήμη. |
| **6. Widget / UI** | Η γενική κλάση βάσης για όλα τα στοιχεία διεπαφής. Εξειδικεύεται σε κλάσεις όπως `Button` και `Slider`. Ο έλεγχος αλληλεπίδρασης γίνεται με τον **έλεγχο περίληψης σημείου σε παραλληλόγραμμο**,. | Πολυμορφισμός UI, Collisions. |
| **7. STL Containers** | Χρήση κατάλληλων δομών (π.χ., `std::vector` για τους κινούμενους επιβάτες/συρμούς, `std::map` για την αποθήκευση κόμβων/σταθμών με βάση το ID τους) για αποδοτική διαχείριση. | Υποχρεωτική Χρήση Συλλογών. |
